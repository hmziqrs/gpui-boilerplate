## Overview

gpui-starter uses a dual i18n system:

- **es-fluent**: Fluent localization with compile-time message validation
- **rust-i18n**: Runtime locale management

Both English (`en`) and Simplified Chinese (`zh-CN`) are included.

## Supported languages

| Code | Language |
|------|----------|
| `en` | English (default) |
| `zh-CN` | 简体中文 |

## Fluent message files

Translation files live in `i18n/<locale>/gpui-starter.ftl`:

```
i18n/
├── en/
│   └── gpui-starter.ftl    # English (68 lines)
└── zh-CN/
    └── gpui-starter.ftl    # Simplified Chinese (68 lines)
```

Example messages from `en/gpui-starter.ftl`:

```ftl
home_title = Welcome to GPUI Starter
home_subtitle = A boilerplate for building desktop apps with GPUI
settings_dark_mode = Dark Mode
settings_language = Language
```

## Adding a new language

1. Create a new directory under `i18n/`:

```bash
mkdir -p i18n/ja
cp i18n/en/gpui-starter.ftl i18n/ja/gpui-starter.ftl
```

2. Translate the messages in the `.ftl` file

3. Add the language variant in `src/app.rs` to the `Languages` enum:

```rust
#[es_fluent_language]
#[derive(Clone, Copy, Debug, EnumIter, EsFluent, PartialEq)]
pub enum Languages {}
```

The `es_fluent_language` macro auto-discovers locales from the `i18n/` directory.

4. Add a button in `src/views/settings.rs` for the new language

5. Rebuild: `cargo run`

## Using translations in code

Use the `localize()` helper from `src/i18n.rs`:

```rust
// Simple message
crate::i18n::localize("home_title", None)

// Message with variables
use std::collections::HashMap;
let mut args = HashMap::new();
args.insert("name", FluentValue::from("GPUI"));
crate::i18n::localize("greeting", Some(&args));
```

For strongly-typed messages (used by form validation), use `localize_message()`:

```rust
crate::i18n::localize_message(&RegistrationFormLabelVariants::Email)
```

## Fluent message format

Fluent supports plurals, gender, and variable interpolation:

```ftl
# Simple
home_title = Welcome to GPUI Starter

# With variable
welcome = Welcome, { $name }!

# Plural
item-count = { $count ->
    [one]   1 item
   *[other] { $count } items
}
```

## Form validation messages

Form validation errors use koruma's fluent integration. The derive macros (`EsFluentVariants`, `KorumaAllFluent`) generate typed message variants that automatically resolve to the current locale:

```rust
#[derive(EsFluentVariants, GpuiForm, Koruma, KorumaAllFluent)]
pub struct RegistrationForm {
    #[koruma(NonEmptyValidation::<_>::builder())]
    pub name: String,
    // ...
}
```

Error messages like "Enter your full name." are looked up from the `.ftl` files based on the current locale.
